# Типичные ошибки новичка на уроке Docker #1

## 1. Путают `image` и `container`

Симптом:

- слушатель говорит "я удалил контейнер, значит удалил образ";
- слушатель не понимает, почему образ остался после `docker rm`.

Что объяснить:

- образ — это шаблон;
- контейнер — это экземпляр образа;
- один образ можно использовать много раз.

## 2. Думают, что `EXPOSE` сам открывает порт наружу

Симптом:

- в `Dockerfile` есть `EXPOSE 80`, но страница не открывается;
- слушатель не использует `-p`.

Что объяснить:

- `EXPOSE` описывает ожидаемый порт контейнера;
- публикация порта наружу делается через `docker run -p host:container`.

## 3. Забывают точку в конце `docker build`

Симптом:

- команда сборки не работает;
- Docker ругается на отсутствие контекста.

Что объяснить:

- точка означает текущую директорию как контекст сборки;
- без контекста Docker не понимает, какие файлы копировать.

## 4. Меняют файл, но не пересобирают образ

Симптом:

- HTML изменили, но в браузере ничего не поменялось;
- слушатель уверен, что Docker "кеширует неправильно".

Что объяснить:

- старый образ сам не обновляется;
- после изменения файлов нужно снова выполнить `docker build`;
- затем нужно запустить контейнер из нового образа.

## 5. Путают имя контейнера и тег образа

Симптом:

- слушатель пытается использовать имя контейнера там, где нужен образ;
- в командах смешиваются `lesson-site` и `lesson-site:1.0`.

Что объяснить:

- тег относится к образу;
- имя относится к контейнеру;
- это разные объекты.

## 6. Пытаются создать контейнер с уже существующим именем

Симптом:

- Docker сообщает, что имя уже используется.

Что объяснить:

- старый контейнер нужно удалить или использовать другое имя;
- полезные команды:

```bash
docker ps -a
docker stop lesson-site
docker rm lesson-site
```

## 7. Открывают не тот порт в браузере

Симптом:

- контейнер запущен, но слушатель смотрит не туда;
- в `docker run` указан `8081:80`, а открывают `8080`.

Что объяснить:

- первый порт в `host:container` — это порт хоста;
- именно его нужно открывать в браузере.

## 8. Не читают текст ошибки

Симптом:

- сразу говорят "Docker не работает";
- не смотрят, что написано в терминале.

Что объяснить:

- большая часть первых ошибок очень буквальная;
- сначала нужно прочитать сообщение полностью;
- затем проверить путь, имя, порт и команду.

## 9. Смотрят только на браузер и забывают про `docker logs`

Симптом:

- приложение не отвечает, но слушатель не смотрит логи.

Что объяснить:

- браузер показывает только внешний результат;
- `docker logs` помогает понять, что происходит внутри контейнера.

## 10. Пугаются маленькой ошибки и бросают задачу

Что важно сказать группе:

- ошибка на первом уроке — это нормальная часть обучения;
- в большинстве случаев проблема в одном маленьком несоответствии;
- Docker на первом уровне гораздо логичнее, чем кажется.
