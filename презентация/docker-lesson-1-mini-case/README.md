# Мини-кейс: найди и исправь ошибку

## Цель

Попробовать собрать образ и самостоятельно найти, почему сборка не проходит.

## Что нужно сделать

1. Выполнить команду сборки:

```bash
docker build -t lesson-case:1.0 .
```

2. Прочитать текст ошибки.
3. Найти, что именно сломано в проекте.
4. Исправить ошибку.
5. Повторить сборку.
6. Запустить контейнер и открыть страницу в браузере.

## Команды после исправления

```bash
docker build -t lesson-case:1.0 .
docker run -d --name lesson-case -p 8082:80 lesson-case:1.0
docker ps
docker logs lesson-case
```

Проверка в браузере:

`http://localhost:8082`

## Подсказка

Сначала внимательно посмотрите на `Dockerfile` и проверьте, существует ли файл, который указан в `COPY`.
