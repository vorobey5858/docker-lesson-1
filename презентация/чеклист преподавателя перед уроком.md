# Чеклист преподавателя перед уроком

## За день до занятия

- Убедиться, что Docker установлен и запускается.
- Проверить команды `docker version` и `docker run hello-world`.
- Убедиться, что образы `hello-world` и `nginx` уже скачаны.
- Проверить, что презентация открывается и используется правильная версия файла.
- Просмотреть папки `docker-lesson-1` и `docker-lesson-1-mini-case`.
- Открыть шпаргалку преподавателя и план урока.
- Решить заранее, будете ли показывать команды как обычный пользователь или через `sudo`.

Команды для проверки:

```bash
docker version
docker info
docker image ls
docker run --rm hello-world
docker pull nginx
```

Если Docker не запущен:

```bash
sudo systemctl status docker --no-pager
sudo systemctl start docker
sudo systemctl enable docker
```

## За 10-15 минут до старта

- Открыть терминал в нужной папке.
- Открыть редактор кода.
- Открыть браузер.
- Проверить, что порт `8080` свободен.
- Убедиться, что Docker daemon запущен.
- Подготовить файл презентации на первом слайде.
- Подготовить резервный план на случай, если у части группы не стартует Docker.

Команды для подготовки окружения:

```bash
cd ~/Документы/презентация
pwd
docker version
docker image ls
sudo systemctl status docker --no-pager
ss -ltnp | grep ':8080 ' || true
```

Если порт `8080` занят, можно быстро посмотреть процесс:

```bash
sudo ss -ltnp | grep ':8080 '
sudo lsof -i :8080
```

## Перед демонстрацией команд

- Очистить старые контейнеры, которые могут мешать.
- Убедиться, что нет контейнера с именем `demo-nginx`.
- Убедиться, что нет контейнера с именем `lesson-site`.
- Проверить, что `localhost:8080` не занят сторонним приложением.

Команды очистки:

```bash
docker ps -a
docker rm -f demo-nginx lesson-site lesson-site-v2 lesson-case 2>/dev/null || true
docker container prune -f
```

Команды проверки:

```bash
docker ps -a
ss -ltnp | grep ':8080 ' || true
```

## Перед основной практикой

- Открыть файл `docker-lesson-1/README.md`.
- Убедиться, что все слушатели видят структуру папки.
- Сразу предупредить про три типовые ошибки:
  - забытая точка в `docker build`;
  - путаница между именем контейнера и тегом образа;
  - изменения в HTML без пересборки образа.

Команды для быстрой самопроверки перед показом:

```bash
cd ~/Документы/презентация/docker-lesson-1
ls -la
ls -la app
cat Dockerfile
```

Если установлен `tree`, можно показать структуру красивее:

```bash
tree -a
```

## Перед мини-кейсом

- Открыть папку `docker-lesson-1-mini-case`.
- Убедиться, что слушатели понимают: ошибка там допущена специально.
- Не называть проблему заранее.
- Дать время на самостоятельную попытку перед общим разбором.

Команды для проверки кейса:

```bash
cd ~/Документы/презентация/docker-lesson-1-mini-case
ls -la
ls -la app
cat Dockerfile
```

## Перед живым показом браузера

- Убедиться, что никакая старая страница не осталась открытой и не вводит группу в заблуждение.
- Проверить, что `localhost:8080` действительно отвечает из контейнера.

Команды для быстрой проверки:

```bash
curl -I http://localhost:8080
docker logs demo-nginx
```

## В конце урока

- Задать 5-7 финальных вопросов группе.
- Проговорить типичные ошибки новичка.
- Выдать домашнее задание в явном виде.
- Напомнить тему следующего занятия.

Команды очистки после урока:

```bash
docker rm -f demo-nginx lesson-site lesson-site-v2 lesson-case 2>/dev/null || true
docker image ls
docker ps -a
```

## Полезная оговорка для Linux-показа

Если вы показываете урок на Linux, лучше заранее использовать один и тот же стиль команд:

- либо везде без `sudo`, если пользователь уже состоит в группе `docker`;
- либо везде с `sudo`, если без него Docker не работает.

Смешивать оба варианта на одном уроке неудобно для новичков.
